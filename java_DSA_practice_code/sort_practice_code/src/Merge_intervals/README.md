# 合并区间 (Merge Intervals)

## 题目描述

以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [starti, endi]`。请你合并所有重叠的区间，并返回一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间。

## 示例

### 示例 1
**输入**: `intervals = [[1,3],[2,6],[8,10],[15,18]]`  
**输出**: `[[1,6],[8,10],[15,18]]`  
**解释**: 区间 `[1,3]` 和 `[2,6]` 重叠，将它们合并为 `[1,6]`。

### 示例 2
**输入**: `intervals = [[1,4],[4,5]]`  
**输出**: `[[1,5]]`  
**解释**: 区间 `[1,4]` 和 `[4,5]` 可被视为重叠区间。

### 示例 3
**输入**: `intervals = [[4,7],[1,4]]`  
**输出**: `[[1,7]]`  
**解释**: 区间 `[1,4]` 和 `[4,7]` 可被视为重叠区间。

## 提示

- `1 <= intervals.length <= 10^4`
- `intervals[i].length == 2`
- `0 <= starti <= endi <= 10^4`

## 解决方案

### 方案一：自定义快速排序 + 结果集合并

**文件**: `Solution.java`

#### 核心思路
1. **排序**: 使用自定义快速排序按照区间的左端点升序排序
2. **合并**: 遍历排序后的区间，维护一个结果列表
   - 如果结果列表为空或当前区间与结果列表中最后一个区间不重叠（前一个的 end < 当前的 start），则直接添加
   - 否则合并两个区间，更新结果列表中最后一个区间的 end 为两者的较大值

#### 代码特点
- 手动实现快速排序，同时交换 `arr` 数组和 `intervals` 二维数组
- 使用 `List<int[]>` 动态存储结果
- 时间复杂度：O(n log n)（排序）+ O(n)（遍历）= O(n log n)
- 空间复杂度：O(log n)（递归栈）+ O(n)（结果集）

#### 注意事项
⚠️ **重要**: main 方法中应打印 merge 方法的**返回值**，而非原数组
```
java
int[][] result = s.merge(intervals);
System.out.println(Arrays.deepToString(result)); // ✓ 正确
// System.out.println(Arrays.deepToString(intervals)); // ✗ 错误
```

### 方案二：内置排序 + 双指针扫描

**文件**: `Solution2.java`

#### 核心思路
1. **排序**: 使用 Java 内置 `Arrays.sort()` 按左端点升序排序
2. **双指针**: 维护当前合并区间的左右边界 `leftLimit` 和 `rightLimit`
   - 遍历后续区间，如果下一个区间的 start ≤ 当前 rightLimit，则可以合并，更新 rightLimit
   - 否则说明无法合并，将当前区间 `[leftLimit, rightLimit]` 加入结果，更新指针为下一个区间
3. **收尾**: 循环结束后，将最后一个区间加入结果

#### 代码特点
- 使用 Java 内置排序，代码更简洁
- 双指针法，逻辑清晰易懂
- 时间复杂度：O(n log n)（排序）+ O(n)（遍历）= O(n log n)
- 空间复杂度：O(n)（结果集）

#### 优势对比
- ✅ 代码更简洁，可读性更好
- ✅ 避免了手动实现排序的复杂性
- ✅ 实际开发中推荐使用此方案

## 两种方案对比

| 特性 | Solution (方案一) | Solution2 (方案二) |
|------|------------------|-------------------|
| 排序方式 | 自定义快速排序 | Java 内置排序 |
| 合并逻辑 | 边遍历边添加到结果集 | 双指针确定完整区间后添加 |
| 代码复杂度 | 较高（需实现排序） | 较低（使用内置 API） |
| 时间复杂度 | O(n log n) | O(n log n) |
| 空间复杂度 | O(n) | O(n) |
| 推荐场景 | 学习排序算法 | 实际项目开发 |
