# 最接近的三数之和 (3Sum Closest)

## 题目描述

给你一个长度为 n 的整数数组 `nums` 和一个目标值 `target`。请你从 `nums` 中选出三个在 **不同下标位置** 的整数，使它们的和与 `target` 最接近。

返回这三个数的和。

假定每组输入只存在恰好一个解。

---

## 示例

### 示例 1

**输入**：
```

nums = [-1,2,1,-4], target = 1
```
**输出**：
```

2
```
**解释**：与 target 最接近的和是 2 (-1 + 2 + 1 = 2)。

---

### 示例 2

**输入**：
```

nums = [0,0,0], target = 1
```
**输出**：
```

0
```
**解释**：与 target 最接近的和是 0（0 + 0 + 0 = 0）。

---

## 约束条件

- `3 <= nums.length <= 1000`
- `-1000 <= nums[i] <= 1000`
- `-10^4 <= target <= 10^4`

---

## 解题思路

### 核心思想：排序 + 双指针

1. **排序**：首先对数组进行升序排序
2. **初始化结果**：用前三个数的和初始化结果
3. **遍历 + 双指针**：
   - 固定一个数 `nums[i]`
   - 使用双指针 `left` 和 `right` 分别指向 `i+1` 和数组末尾
   - 计算三数之和 `sum = nums[i] + nums[left] + nums[right]`
   - 比较 `|sum - target|` 和 `|result - target|`，如果前者更小则更新结果
4. **指针移动策略**：
   - 如果 `sum > target`，则 `right--`（需要更小的和）
   - 如果 `sum < target`，则 `left++`（需要更大的和）
   - 如果 `sum == target`，直接返回 `sum`（已经找到最接近的）

### 优化点

- **去重处理**：如果 `nums[i] == nums[i-1]`，跳过当前循环，避免重复计算
- **提前终止**：当 `sum == target` 时，直接返回，无需继续查找

---

## 复杂度分析

- **时间复杂度**：O(n²)
  - 排序：O(n log n)
  - 双指针遍历：O(n²)
  - 总体：O(n²)

- **空间复杂度**：O(1)
  - 只需要常数级别的额外空间
