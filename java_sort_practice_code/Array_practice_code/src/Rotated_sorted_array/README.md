# 搜索旋转排序数组 (Search in Rotated Sorted Array)

## 题目描述

整数数组 nums 按升序排列，数组中的值 互不相同 。

在传递给函数之前，nums 在预先未知的某个下标 k（0 <= k < nums.length）上进行了 向左旋转，使数组变为 [nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]（下标 从 0 开始 计数）。例如， [0,1,2,4,5,6,7] 下标 3 上向左旋转后可能变为 [4,5,6,7,0,1,2] 。

给你 旋转后 的数组 nums 和一个整数 target ，如果 nums 中存在这个目标值 target ，则返回它的下标，否则返回 -1 。

你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

---

## 示例

### 示例 1

**输入：** `nums = [4,5,6,7,0,1,2], target = 0`  
**输出：** `4`

### 示例 2

**输入：** `nums = [4,5,6,7,0,1,2], target = 3`  
**输出：** `-1`

### 示例 3

**输入：** `nums = [1], target = 0`  
**输出：** `-1`

---

## 提示

- `1 <= nums.length <= 5000`
- `-10^4 <= nums[i] <= 10^4`
- `nums` 中的每个值都 独一无二
- 题目数据保证 nums 在预先未知的某个下标上进行了旋转
- `-10^4 <= target <= 10^4`

---

## 解题思路

### 方法一：二分查找法 ⭐

**核心思想：**

虽然数组被旋转了，但仍然可以利用二分查找的思想。每次二分后，至少有一半是有序的，通过判断 target 是否在有序的那一半中来决定搜索范围。

**算法步骤：**

1. 初始化左右指针 `left = 0`, `right = n-1`
2. 当 `left <= right` 时循环：
    - 计算中间位置 `mid = left + (right - left) / 2`
    - 如果 `nums[mid] == target`，返回 `mid`
    - 判断哪一半是有序的：
        - 如果 `nums[left] <= nums[mid]`，说明左半部分有序
            - 若 `target` 在左半部分范围内，则 `right = mid - 1`
            - 否则 `left = mid + 1`
        - 否则右半部分有序
            - 若 `target` 在右半部分范围内，则 `left = mid + 1`
            - 否则 `right = mid - 1`
3. 未找到则返回 `-1`

**时间复杂度：** O(log n)  
**空间复杂度：** O(1)
