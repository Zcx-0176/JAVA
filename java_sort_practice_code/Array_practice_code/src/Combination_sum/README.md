# 组合总和 (Combination Sum)

## 题目描述

给你一个 **无重复元素** 的整数数组 `candidates` 和一个目标整数 `target` ，找出 `candidates` 中可以使数字和为目标数 `target` 的 **所有不同组合** ，并以列表形式返回。你可以按 **任意顺序** 返回这些组合。

`candidates` 中的 **同一个** 数字可以 **无限制重复被选取** 。如果至少一个数字的被选数量不同，则两种组合是不同的。

对于给定的输入，保证和为 `target` 的不同组合数少于 150 个。

---

## 示例

### 示例 1

**输入：** `candidates = [2,3,6,7], target = 7`  
**输出：** `[[2,2,3],[7]]`  
**解释：** 
- 2 和 3 可以形成一组候选，2 + 2 + 3 = 7 。注意 2 可以使用多次。
- 7 也是一个候选， 7 = 7 。
- 仅有这两种组合。

---

### 示例 2

**输入：** `candidates = [2,3,5], target = 8`  
**输出：** `[[2,2,2,2],[2,3,3],[3,5]]`

---

### 示例 3

**输入：** `candidates = [2], target = 1`  
**输出：** `[]`

---

## 约束条件

- `1 <= candidates.length <= 30`
- `2 <= candidates[i] <= 40`
- `candidates` 的所有元素 **互不相同**
- `1 <= target <= 40`

---

## 解题思路

### 方法一：回溯算法（DFS）⭐⭐⭐

使用深度优先搜索和回溯算法，枚举所有可能的组合。

**核心思想：**
- 从数组起始位置开始，每次选择一个数字加入当前组合
- **关键点**：允许重复选择同一个数字，因此递归时传递当前索引 `i` 而不是 `i+1`
- 当总和等于 target 时，找到一个有效组合
- 当总和超过 target 时，剪枝返回

**算法步骤：**
1. 对数组排序（便于剪枝优化）
2. 定义回溯函数 `backtrack(index, sum, currentCombination)`
   - `index`: 当前可以选择的起始位置
   - `sum`: 当前组合的总和
   - `currentCombination`: 当前的组合列表
3. 终止条件：
   - `sum == target`: 找到有效组合，加入结果集
   - `sum > target`: 超过目标值，剪枝
4. 遍历从 `index` 开始的所有数字：
   - 选择当前数字，加入组合
   - 递归调用（注意传递 `i` 而不是 `i+1`，因为可以重复选择）
   - 撤销选择（回溯）

**时间复杂度：** O(S)，其中 S 为所有可行解的长度之和  
**空间复杂度：** O(target/min(candidates))，递归栈的深度

### 关键代码解析

```
java
// 为什么主函数不需要循环？
public List<List<Integer>> combinationSum(int[] candidates, int target) {
backtrack(candidates, 0, 0, new ArrayList<>(), target, result);
//                    ↑ 从索引 0 开始，递归内部会自动遍历所有可能
}

// 为什么递归中可以重复选择？
for(int i = index; i < candidates.length; i++) {
backtrack(candidates, i, ...);  // ← 传递 i，不是 i+1
//                        允许下一次递归继续选择当前元素
}

// 为什么要用 break 而不是 return？
if(sum + candidates[i] > target) {
break;  // 数组已排序，后面的数字更大，直接跳出循环即可
}
```

## 注意事项

1. **可重复选择**：这是与后续变体题目的关键区别，递归时传递 `i` 而非 `i+1`
2. **无重复元素**：题目保证 `candidates` 中元素互不相同，简化了去重逻辑
3. **剪枝优化**：排序后，当发现当前和超过 target 时可以提前终止
4. **回溯恢复现场**：递归返回后记得移除最后加入的元素，恢复 sum 值
5. **深拷贝结果**：添加到结果集时要 `new ArrayList<>(temp)`，避免引用问题
