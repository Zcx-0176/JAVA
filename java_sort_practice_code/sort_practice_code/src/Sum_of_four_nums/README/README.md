# 四数之和 (4Sum)

## 题目描述

给你一个由 `n` 个整数组成的数组 `nums`，和一个目标值 `target`。请你找出并返回满足下述全部条件且不重复的四元组 `[nums[a], nums[b], nums[c], nums[d]]`：

- `0 <= a, b, c, d < n`
- `a`、`b`、`c` 和 `d` 互不相同
- `nums[a] + nums[b] + nums[c] + nums[d] == target`

你可以按 **任意顺序** 返回答案。

**注意**：若两个四元组元素一一对应，则认为两个四元组重复，答案中不可以包含重复的四元组。

---

## 示例

### 示例 1

**输入**：
```

nums = [1,0,-1,0,-2,2], target = 0
```
**输出**：
```

[[-2,-1,1,2],[-2,0,0,2],[-1,0,0,1]]
```
---

### 示例 2

**输入**：
```

nums = [2,2,2,2,2], target = 8
```
**输出**：
```

[[2,2,2,2]]
```
---

## 约束条件

- `1 <= nums.length <= 200`
- `-10^9 <= nums[i] <= 10^9`
- `-10^9 <= target <= 10^9`

---

## 解题思路提示

1. **排序 + 双指针**：先排序，然后固定两个数，用双指针找另外两个数
2. **两层循环 + 双指针**：
   - 第一层循环固定第一个数 `nums[i]`
   - 第二层循环固定第二个数 `nums[j]`
   - 使用双指针 `left` 和 `right` 找第三和第四个数
3. **去重技巧**：
   - 在第一层循环中跳过重复的 `nums[i]`
   - 在第二层循环中跳过重复的 `nums[j]`
   - 在双指针移动时跳过重复的元素
4. **剪枝优化**：
   - 如果当前最小的四个数之和大于 `target`，可以直接返回（因为后面的和只会更大）
   - 如果当前最大的四个数之和小于 `target`，可以跳过本次循环（因为当前的和已经是最小的了）
5. **时间复杂度**：O(n³)
6. **空间复杂度**：O(1)（不计返回值的空间）
